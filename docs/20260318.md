## Problemas Detectados

### 1. Columna `updated_at` faltante
- **Archivo:** `migrations/001_init.sql`
- **Problema:** El código intentaba actualizar columna `updated_at` que no existía en el schema
- **Error:** `no such column: updated_at`
- **Solución:** Agregar `updated_at TIMESTAMP` a la tabla jobs

### 2. `retry_count` no se persistía
- **Archivo:** `src/database.py`
- **Problema:** `update_job_status()` no aceptaba parámetro `retry_count`
- **Solución:** Agregar parámetro opcional `retry_count: Optional[int]`

### 3. `complete_job()` no pasaba `retry_count` a la DB
- **Archivo:** `src/job_queue.py`
- **Problema:** Se calculaba `retry_count` pero nunca se guardaba
- **Solución:** Pasar `retry_count` a `update_job_status()`

### 4. Conexión DB cerrada prematuramente
- **Archivo:** `src/job_queue.py` línea 512
- **Problema:** `_get_all_jobs_status()` cerraba conexión compartida con `conn.close()`
- **Error:** `Cannot operate on a closed database`
- **Solución:** Eliminar `conn.close()` (la conexión es manejada por Database)

### 5. Test de retry con lógica incorrecta
- **Archivo:** `tests/audit/cold_test_phase1.py`
- **Problema:** El test usaba `dequeue()` en un loop, pero `dequeue()` devuelve jobs en round-robin
- **Solución:** Llamar `complete_job()` directamente sin `dequeue()`

### 6. Test de callbacks reutilizaba job corrupto
- **Archivo:** `tests/audit/cold_test_phase1.py`
- **Problema:** Reutilizaba `jid2` que ya había sido dequeued
- **Solución:** Crear job fresco `jid_callback` para el test
