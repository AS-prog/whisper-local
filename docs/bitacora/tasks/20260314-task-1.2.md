## Especificación Técnica

### Estados de Job

```
pending → processing → completed
   ↓         ↓
   └─────> failed (después de 3 reintentos)
```

### Lógica de Priorización

1. **Cola `pending`**: Archivos ≤ 40MB, prioridad alta
2. **Cola `big_files`**: Archivos > 40MB, prioridad baja

**Algoritmo de Dequeue:**
1. Siempre intentar tomar de `pending` primero
2. Si `big_files` tiene > 5 archivos, aplicar intercalación:
   - Por cada 3 archivos de `pending`, tomar 1 de `big_files`
3. Si no hay pendientes en `pending`, tomar de `big_files`
4. Orden FIFO (created_at ASC) dentro de cada cola

### Sistema de Reintentos

**Condiciones para reintentar:**
- Job en estado `failed`
- `retry_count < 3`
- Tiempo transcurrido desde último intento ≥ `5 min * retry_count`

**Proceso:**
1. `process_retries()` identifica jobs elegibles
2. Actualiza estado a `pending`
3. Incrementa `retry_count`
4. Job vuelve a la cola para procesamiento

### Callbacks de Progreso

**Registro:**
```python
def my_callback(job_id: int, progress: int, status: str) -> None:
    print(f"Job {job_id}: {progress}% - {status}")

queue.register_progress_callback(job_id, my_callback)
```

**Notificación:**
- Llamar `notify_progress(job_id, progress)` durante procesamiento
- Callbacks se invocan sincrónicamente (thread-safe con Lock)
- Callbacks se limpian automáticamente al completar job

### Ejemplo de Uso

```python
from src.database import Database
from src.job_queue import JobQueue

# Inicializar
db = Database("whisper.db")
queue = JobQueue(db)

# Encolar archivo
job_id = queue.enqueue(
    file_path="inputs/audio.mp3",
    user_id="user_123",
    platform="telegram"
)

# Registrar callback de progreso
def on_progress(job_id, progress, status):
    print(f"Progreso: {progress}%")

queue.register_progress_callback(job_id, on_progress)

# Procesar siguiente job
job = queue.dequeue()
if job:
    # Procesar archivo...
    queue.notify_progress(job['id'], 50)
    
    # Completar exitosamente
    queue.complete_job(job['id'], success=True)
    
    # O marcar como fallido (se reintentará automáticamente)
    # queue.complete_job(job['id'], success=False, error="Timeout")

# Procesar reintentos
queue.process_retries()

# Ver estado de colas
status = queue.get_queue_status()
print(f"Pending: {status['pending_count']}")
print(f"Big Files: {status['big_files_count']}")
print(f"Processing: {status['processing_count']}")
```

### Manejo de Errores

- **Enqueue fallido**: Retorna -1, log de error
- **Dequeue vacío**: Retorna None
- **Job no encontrado**: Log de warning, continúa operación
- **Callback no registrado**: Silencioso (notificación sin efecto)

### Thread Safety

El sistema utiliza `threading.Lock` para proteger:
- Registro de callbacks (`_progress_callbacks`)
- Notificación de progreso

Múltiples threads pueden:
- Llamar `enqueue()` simultáneamente
- Llamar `dequeue()` simultáneamente
- Registrar callbacks simultáneamente

### Notas de Implementación

1. **Determinación de cola**: Archivos > 40MB van a `big_files`, resto a `pending`
2. **Intercalación**: Previene starvation de archivos grandes
3. **Reintentos**: Basados en timestamp de última actualización
4. **Callbacks**: Diccionario por job_id, lista de funciones
5. **FIFO**: Ordenamiento por `created_at` ASC en consultas SQL
6. **Transacciones**: Cada operación hace commit explícito via Database
