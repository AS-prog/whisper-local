## Especificación Técnica

### Schema de Base de Datos

El módulo gestiona las siguientes tablas (definidas en `migrations/001_init.sql`):

#### Tabla `jobs`
```sql
CREATE TABLE jobs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path TEXT NOT NULL,
    file_hash TEXT,
    file_size_mb REAL,
    status TEXT DEFAULT 'pending',
    queue TEXT DEFAULT 'pending',
    user_id TEXT,
    platform TEXT DEFAULT 'cli',
    progress INTEGER DEFAULT 0,
    retry_count INTEGER DEFAULT 0,
    error_message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### Tabla `transcription_cache`
```sql
CREATE TABLE transcription_cache (
    file_hash TEXT PRIMARY KEY,
    transcription TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### Tabla `rate_limits`
```sql
CREATE TABLE rate_limits (
    user_id TEXT,
    platform TEXT,
    request_count INTEGER DEFAULT 0,
    window_start TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, platform)
);
```

#### Tabla `metrics`
```sql
CREATE TABLE metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    metric_name TEXT NOT NULL,
    metric_value REAL NOT NULL,
    recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Ejemplo de Uso

```python
from src.database import Database

# Inicializar base de datos
db = Database("whisper.db")

# Crear un job
job_id = db.create_job(
    file_path="inputs/audio.mp3",
    file_hash="abc123...",
    file_size_mb=15.5,
    queue="pending",
    user_id="user_123",
    platform="telegram"
)

# Obtener job
job = db.get_job(job_id)
print(f"Status: {job['status']}")

# Actualizar progreso
db.update_job_progress(job_id, progress=50, chunks_completed=2)

# Verificar rate limit
if db.check_rate_limit("user_123", "telegram", max_per_hour=5):
    print("Puede procesar")
else:
    print("Límite alcanzado")

# Cerrar conexión
db.close()
```

### Manejo de Errores

Todos los métodos incluyen manejo de excepciones:
- `sqlite3.Error`: Errores de base de datos (log y retorno de valores por defecto)
- `FileNotFoundError`: Archivo de migración no encontrado
- Validación de parámetros: Retorno de `None` o `-1` en caso de error

### Dependencias

- `sqlite3` (stdlib): Base de datos SQLite
- `logging` (stdlib): Logging de operaciones
- `datetime` (stdlib): Timestamps
- `typing` (stdlib): Type hints

### Notas de Implementación

1. **Conexión persistente**: La conexión se mantiene abierta durante la vida del objeto `Database`
2. **Row factory**: Configurada como `sqlite3.Row` para acceso tipo diccionario
3. **Transacciones**: Cada operación hace `commit` explícito
4. **Schema migrations**: El schema se carga desde archivo SQL externo
5. **Rate limiting**: Ventana de 1 hora, contador por usuario+plataforma
6. **Cache**: Sin TTL automático, requiere llamada explícita a `clean_old_cache()`
